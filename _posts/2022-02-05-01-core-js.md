## 4. 기본형 데이터와 참조형 데이터

### 4-1. 불변값

변수와 상수의 구분점은 '변경 가능성'이다. 변경 가능하면 변수, 변경 불가능 하면 상수다. 하지만 상수와 불변값은 다르다. 변수와 상수를 구분할 때 변경 가능성의 대상은 **변수 영역**의 메모리이다. 한 번 데이터 할당이 이루어진 변수 공간에 다른 데이터를 재할당할 수 있는지 여부가 관건이다. 하지만 불변성 여부를 구분할 때 변경 가능성의 대상은 **데이터 영역**의 메모리이다.
<BR>

```javascript
var a = "abc";
a = a + "def";

var b = 5;
var c = 5;
var b = 7;
```

기본형 데이터인 숫자, 문자열, boolean, null, undefined, Symbol은 모두 불변값이다. 위는 그 중 숫자와 문자열에 대한 예이다.<BR><BR>
위에서 'abc'와 'abcdef'는 완전히 별개의 데이터이다(이전 글 참조). 4번째 줄에서 변수 b에 숫자 5를 할당했을 때 컴퓨터는 우선 데이터 영억에서 5를 찾고 없다면 데이터 공간을 하나 만들어 저장하고 그 주소를 b에 저장한다. 5번째 줄에서 똑같이 5를 할당하려고 할 때 컴퓨터는 데이터 영역에서 이미 만들어 놓은 5를 찾아 그 주소값을 재활용한다.<BR><BR>
6번째 줄에서는 변수 b의 값을 7로 바꾸고자 한다. 그러면 데이터 영역에 저장된 5를 7이라는 데이터로 바꾸는 것이 아니라, 데이터 영역에서 7을 찾아 그 주소값을 b에 저장한다. 즉 5와 7 모두 다른 값으로 변경할 수 없다. 즉 숫자도 문자열도 변경은 새로 만드는 동작을 통해서만 이루어진다. 한 번 만들어지면 가비지 컬렉팅을 당하지 않는 한 영원히 변하지 않는 것이 불변값이다.
<BR>
<BR>

### 4-2. 가변값

참조형 데이터는 기본적으로 가변값인 경우가 많지만 설정에 따라 변경 불가능한 경우도 있고, 아예 불변값으로 활용하는 방안도 있지만 이건 예외적인 경우다. 참조형 데이터는 변수에 할당하는 과정이 조금 상이하다.<BR>

```javascript
var obj1 = {
  a: 1,
  b: "bbb",
};
```

컴퓨터는 우선 변수 영역의 빈 공간(임의로 1002)을 확보하고 그 주소의 식별자를 obj1이라고 지정한다. 임의의 데이터 저장공간(5001)에 데이터를 저장하고 주소값만 연결하려고 보니 그 데이터가 여러 개의 프로퍼티로 이루어진 데이터 그룹이다. 이 그룹 내부의 프로퍼티를 저장하기 위해 별도로 변수 영역을 마련하고, 그 영역의 주소를 5001에 저장한다. 그 변수 영역의 이름은 각 프로퍼티 즉 a와 b라고 한다. 데이터 영역에서 a에 할당된 1을 찾고, 그 주소를 5001의 변수 영역에 저장한다. b도 마찬가지의 과정을 거친다.<BR><BR>
즉 기본형 데이터는 변수 영역에 데이터 영역의 주소값이 바로 저장됐다면 이런 타입의 참조형 데이터는 *객체의 변수(프로퍼티) 영역*이 별도로 존재해 데이터 영역에 저장된다는 것이다.<BR><BR>
이때 데이터 영역은 기존의 메모리 공간을 그대로 활용하고 있고, 데이터 영역에 저장된 값은 모두 불변값이다. 그러나 객체 영역에 있는 프로퍼티의 주소값은 프로퍼티의 데이터가 바뀜에 따라 얼마든지 바뀔 수 있다. 그러므로 참조형 데이터는 보통 불변하지 않다고 하는 것이다.
<BR>
<BR>

```javascript
var obj1 = {
  a: 1,
  b: "bbb",
};
obj1.a = 2;
```

이 때 프로퍼티 a에 숫자 2를 할당하려고 하면, 데이터 영역에서 2를 찾아 그 주소값을 객체의 변수 영역에 저장한다. 4번째 줄의 명령 전과 후에 변수 obj1이 바라보고 있는 주소는 5001로 변하지 않았다. 문자열이나 숫자를 재할당 했을 때 바라보고 있던 주소가 바뀐 것과 달리 별개의 데이터가 만들어진 것이 아니라 객체 내부의 값만 바뀐 것이다.
<BR>
<BR>
그렇다면 중첩 객체의 경우 어떻게 될까?<BR>

```javascript
var obj = {
  x: 3,
  arr: [3, 4, 5],
};
```

컴퓨터는 우선 변수 영역의 빈 공간 1002를 확보하고 그 식별자를 obj로 지정한다. 그리고 데이터를 5001이라는 임의의 공간에 저장하려는데 이 데이터가 객체다. 이 객체의 각 변수(프로퍼티)들을 별도의 변수 영역 7103과 7104를 마련해 각각 x와 arr을 저장하고, 그 주소를 5001에 저장한다. 7104에 저장할 값은 배열이므로 역시 데이터 그룹이다. 이 내부 프로터피를 저장하기 위해 별도의 변수 영역 8104, 8105, 8106을 저장하고 이 주소값들을 또 별도의 공간 5003에 저장한 다음 5003을 arr이 식별자인 7104에 저장한다.<BR><BR>
즉 1002(식별자 obj)에서 5001이라는 주소값을 보고 해당 공간을 봤더니 그 곳에 7103(식별자 x), 7104(식별자 arr)이라는 데이터가 있다. 7104 주소로 찾아갔더니 그 데이터는 또 5003(식별자 arr)이라는 주소값이고 그곳에는 8104, 8105, 8106이라는 세 개의 주소값이 저장되어 있다. 각 주소로 찾아가면 각 데이터에 해당하는 주소값이 있고(3은 데이터 영역 5002, 4는 5004, 5는 5005 이런 식) 이 주소값으로 데이터 영역에서 검색해 데이터를 구할 수 있다. 이렇게 적으니 굉장히 복잡해 보이지만 이름을 가진 상자를 열었더니 상자가 또 있고 라는 식으로 이해하면 어떨까? 마트료시카처럼 말이다.<BR><BR>
이때 arr의 데이터인 배열의 각 인덱스가 8104, 8105, 8106에 저장되어 있는 것이고 그 영역의 식별자는 해당 값의 인덱스이다. 이제 obj.arr[1]을 검색하려 하면 메모리에서는 다음과 같은 과정을 거치게 된다.<BR><BR>

1. obj 검색1 - obj라는 식별자를 가진 주소를 찾는다(1002)
2. obj 검색2 - 값이 주소이므로 그 주소로 이동(5001)
3. obj 검색3 - 값이 주소이므로 그 주소로 이동(7103, 7104)
4. obj.arr 검색1 - arr이라는 식별자를 가진 주소를 찾음(7104)
5. obj.arr 검색2 - 값이 주소이므로 그 주소로 이동(5003)
6. obj.arr 검색3 - 값이 주소이므로 그 주소로 이동(8104, 8105, 8106)
7. obj.arr[1] 검색1 - 인덱스 1에 해당하는 주소를 찾는다(8105)
8. obj.arr[1] 검색2 - 값이 주소이므로 그 주소로 이동(5004)
9. obj.arr[1] 검색3 - 값이 숫자형 데이터이므로 4를 반환
   <BR>
   만약 이 상태에서 다음과 같이 재할당을 내린다면 어떻게 될까?

```javascript
obj.arr = "str";
```

새로운 데이터 영역 5006에 문자열 'str'을 저장하고 그 주소를 7104에 저장한다. 그러면 배열 [3,4,5]가 저장된 5003은 자신의 주소를 참조하는 변수가 하나도 없게 된다. 데이터의 주소를 참조하는 변수의 개수를 참조 카운트라고 하는데, arr: [3,4,5] 였을 때 5003의 참조 카운트는 1이었다가 7104(식별자 arr)에 'str'의 주소값인 5006이 저장되는 순간 0이 된다. 참조 카운트가 0인 메모리 주소는 가비지 컬렉터(GC)의 수거 대상이 된다. 수거된 메모리는 다시 새로운 값을 할당할 수 있는 빈 공간이 된다.
<BR>
<BR>

### 4-3. 변수 복사 비교

```javascript
var a = 10;
var b = a;

var obj1 = { c: 10, d: "ddd" };
var obj2 = obj1;
```

본격적으로 기본형 데이터와 참조형 데이터의 차이를 알아보자.
<BR><BR>
첫번째 행에서 변수 영역의 빈 공간 1001을 확보하고 식별자를 a로 지정한 다음, 데이터 영역의 빈 공간 5001에 데이터 10을 저장하고 a의 데이터에 주소값 5001을 저장한다.<BR><BR>
두번째 행에서는 변수 영역의 빈 공간 1002를 확보해 식별자를 b로 지정한 다음 식별자 a를 검색해 그 값을 찾아와야 한다. 1001에 저장된 값인 5001을 1002에 대입하는 것이다. 이것이 기본형 데이터의 복사다.<BR><BR>
참조형 데이터의 경우, 네번째 행에서 우선 변수 영역의 빈 공간 1003을 확보해 식별자를 obj로 지정한다. 그리고 데이터 영역의 빈 공간 5002를 확보하고, 여기에 데이터 그룹이 담겨야 하므로 별도의 변수 영역 7103과 7104를 확보해 식별자를 각각 c와 d로 지정한 다름 7103과 7104 주소값을 5002에 저장한다. 그리고 c의 데이터인 10은 이미 5001에 있으므로 이 주소를 7103에 저장하고, d의 값인 'ddd'의 경우 아직 없기 때문에 새 데이터 영역 5003을 확보해 저장하고 그 주소를 7104에 연결한다. 이렇게 객체에 대한 변수 선언과 할당이 끝난다.<BR><BR>
다섯번째 행에서 복사가 이루어지면 우선 새 변수 영역 1004의 식별자를 obj2로 지정한 다음, 식별자 obj1을 검색해(1003 영역) 그 값인 5002라는 주소값을 1004의 값으로 대입한다. 여기까지는 기본형 데이터와 참조형 데이터 모두 같은 주소를 바라보게 된다는 점이 동일하다. 1001(a)와 1002(b)는 모두 데이터 영역 5001을 바라고보, 1003(obj1)과 1004(obj2)는 모두 5002를 값으로 가진다. 하지만 데이터 할당 과정에서 다른점이 있기 때문에 복사 이후의 동작에서는 차이가 발생한다.<BR>

```javascript
var a = 10;
var b = a;

var obj1 = { c: 10, d: "ddd" };
var obj2 = obj1;

b = 15;
obj2.c = 20;
```

b에 새로운 값을 할당하면(15가 데이터 영역에 없는 경우), 새 공간 5004에 데이터를 저장하고 b가 바라보는 주소가 5004로 바뀐다.<BR><BR>
그 다음 줄에서 obj2.c에 새로운 값을 할당하면 새로운 공간 5005에 20을 저장한 다음 변수 영역에서 obj2를 찾고(1004), 그 값인 5002가 가리키는 변수 영역에서 다시 c를 찾아(7103) 그곳에 5005를 대입한다.<BR><BR>
즉 기본형 데이터는 b를 식별자로 가지는 1002의 값이 바로 변경된 반면 참조형 데이터의 경우 obj2를 식별자로 가지는 1004의 값은 달라지지 않았다. 1004의 값의 값인 7103의 값이 바뀌었다.<BR><BR>
불변성을 논할 때는 데이터 영역이 그 기준이 된다. 기본형 데이터는 변수 영역의 값이 바뀌었지만(변수영역에 저장된 주소값) 그 값으로 찾은 데이터 영역의 값은 바뀌지 않았다. 그저 복사해서 같은 주소값을 바라보던 a와 b가 이제 서로 다른 주소를 바라보게 된 것 뿐이다.<BR><BR>
그러나 참조형 데이터의 경우 변수 영역(1004)의 값(5002)은 바뀌지 않아 복사 후 재할당을 했음에도 불구하고 변수 obj1과 obj2는 여전히 5002라는 같은 주소를 바라보고 있다. 그런데 만약 obj2의 프로퍼티 c에 재할당을 하는 것이 아니라 객체 자체를 재할당 한다면 어떻게 될까?

```javascript
var a = 10;
var b = a;

var obj1 = { c: 10, d: "ddd" };
var obj2 = obj1;

b = 15;
// obj2.c = 20;
obj2 = { c: 20, d: "ddd" };
```

이렇게 하면 메모리 데이터 영역에 새 객체가 저장되고 그 위치가 변수 영역의 obj2 위치에 저장된다. 똑같이 c의 값이 20이 된 것인데도 이전 예시와 달리 obj2를 식별자로 가지는 변수 영역의 값만 변경되는 것이다.<BR><BR>
즉 참조형 데이터가 가변값이라는 명제는 객체 자체를 변경하는 경우가 아니라 그 내부의 프로퍼티를 변경할 때만 성립한다.
