## 3. LexicalEnvironment

<BR>

### 3-1. environmentRecord와 호이스팅

<BR>

_environmentRecord_

현재 컨텍스트와 관련된 코드의 식별자 정보(컨텍스트를 구성하는 함수에 지정된 매개변수 식별자, 선언한 함수가 있다면 그 자체, var로 선언도니 변수의 식별자 등)이 저장됨. 컨텍스트 내부에서 순서대로 수집.

 <BR>

_호이스팅 규칙_

코드가 실제로 실행되기 전에 실행 컨텍스트의 변수 정보는 모두 수집된 상태이므로, 자바스크립트 엔진은 이미 해당 환경의 변수명들을 알고 있다. 호이스팅이란 실제로 자바스크립트 엔진이 변수 선언을 위로 끌어올린 것은 아니지만 그와 다름없이 간주하는 개념이다.

```javascript
function a(x) {
  console.log(x); // 1
  var x;
  console.log(x); // 1
  var x = 2;
  console.log(x); // 2
}
a(1);
```

위 코드에서 주목해야 할 점은 var x; 바로 직후에 콘솔을 출력했을 때 undefined가 아닌 1이 출력된다는 것이다. 함수 a가 인자로 받는 x는 함수 내부에서 x라는 변수를 선언하는 것과 LexicalEnvironment 입장에서는 다를 바가 없으므로 다음과 같이 볼 수도 있다.

```javascript
function a() {
  var x = 1;
  console.log(x);
  var x;
  console.log(x);
  var x = 2;
  console.log(x);
}
a();
```

이 상태에서 호이스팅이 적용된다고 하면 즉, 이미 자바스크립트 엔진이 변수명을 모두 알고 있어 변수 선언이 맨 위로 끌어올려지고 할당만 원래 자리에서 이루어진다고 가정하면 실제와는 다르지만 이렇게 이해해볼 수 있다.

```javascript
function a() {
  var x;
  var x;
  var x;

  x = 1;
  console.log(x);
  console.log(x);
  x = 2;
  console.log(x);
}
a();
```

<BR>
함수 선언을 추가한 경우를 살펴보자

```javascript
function a() {
  console.log(b); // function b() {}
  var b = "bbb";
  console.log(b); // bbb
  function b() {}
  console.log(b); // bbb
}
a();
```

a 함수가 실행되는 순간 a 함수의 실행 컨텍스트가 생성된다. 이떠ㅐ 변수명과 함수 선언의 정보를 수집한다(위로 끌어올린다고 간주). 변수는 선언부만 끝어올리고 할당하는 부분은 제자리에 있는 반면 함수 선언은 그 전체를 끌어올린다. 호이스팅을 마쳤다고 가정했을 때, 실제로는 다르겠지만 다음과 같이 볼 수 있을 것이다.

```javascript
function a() {
  var b; // 변수는 선언부만 끌어올린다
  function b() {} // 함수 선언은 전체를 끌어올린다

  console.log(b);
  b = "bbb";
  console.log(b);
  console.log(b);
}
a();
```
