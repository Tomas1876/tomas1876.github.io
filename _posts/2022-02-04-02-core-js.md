## 4. 기본형 데이터와 참조형 데이터

### 4-1. 불변값

변수와 상수의 구분점은 '변경 가능성'이다. 변경 가능하면 변수, 변경 불가능 하면 상수다. 하지만 상수와 불변값은 다르다. 변수와 상수를 구분할 때 변경 가능성의 대상은 **변수 영역**의 메모리이다. 한 번 데이터 할당이 이루어진 변수 공간에 다른 데이터를 재할당할 수 있는지 여부바 관건이다. 하지만 불변성 여부를 구분할 때 변경 가능성의 대상은 **데이터 영역**의 메모리이다.
<BR>

```
01 var a = 'abc';
02 a = a + 'def';
03
04 var b = 5;
05 var c= 5;
06 var b = 7;
```

기본형 데이터인 숫자, 문자열, boolean, null, undefined, Symbol은 모두 불변값이다. 위는 그 중 숫자와 문자열에 대한 예이다.<BR>
위에서 'abc'와 'abcdef'는 완전히 별개의 데이터이다(이전 글 참조). 4번째 줄에서 변수 b에 숫자 5를 할당했을 때 컴퓨터는 우선 데이터 영억에서 5를 찾고 없다면 데이터 공간을 하나 만들어 저장하고 그 주소를 b에 저장한다. 5번째 줄에서 똑같이 5를 할당하려고 할 때 컴퓨터는 데이터 영역에서 이미 만들어 놓은 5를 찾아 그 주소값을 재활용한다.<BR>
6번째 줄에서는 변수 b의 값을 7로 바꾸고자 한다. 그러면 데이터 영역에 저장된 5를 7이라는 데이터로 바꾸는 것이 아니라, 데이터 영역에서 7을 찾아 그 주소값을 b에 저장한다. 즉 5와 7 모두 다른 값으로 변경할 수 없다. 즉 숫자도 문자열도 변경은 새로 만드는 동작을 통해서만 이루어진다. 한 번 만들어지면 가비지 컬렉팅을 당하지 않는 한 영원히 변하지 않는 것이 불변값이다.
<BR>
<BR>

### 4-2. 가변값

참조형 데이터는 기본적으로 가변값인 경우가 많지만 설정에 따라 변경 불가능한 경우도 있고, 아예 불변값으로 활용하는 방안도 있지만 이건 예외적인 경우다. 참조형 데이터는 변수에 할당하는 과정이 조금 상이하다.

```
01 obj1 = {
02    a : 1,
03    b: 'bbb'
04 };
```

컴퓨터는 우선 변수 영역의 빈 공간(임의로 1002)을 확보하고 그 주소의 식별자를 obj1이라고 지정한다. 임의의 데이터 저장공간(5001)에 데이터를 저장하고 주소값만 연결하려고 보니 그 데이터가 여러 개의 프로퍼티로 이루어진 데이터 그룹이다. 이 그룹 내부의 프로퍼티를 저장하기 위해 별도로 변수 영역을 마련하고, 그 영역의 주소를 5001에 저장한다. 그 변수 영역의 이름은 각 프로퍼티 즉 a와 b라고 한다. 데이터 영역에서 a에 할당된 1을 찾고, 그 주소를 5001의 변수 영역에 저장한다. b도 마찬가지의 과정을 거친다.<BR>
즉 기본형 데이터는 변수 영역에 데이터 영역의 주소값이 바로 저장됐다면 이런 타입의 참조형 데이터는 *객체의 변수(프로퍼티) 영역*이 별도로 존재해 데이터 영역에 저장된다는 것이다.<BR>
이때 데이터 영역은 기존의 메모리 공간을 그대로 활용하고 있고, 데이터 영역에 저장된 값은 모두 불변값이다. 그러나 객체 영역에 있는 프로퍼티의 주소값은 프로퍼티의 데이터가 바뀜에 따라 얼마든지 바뀔 수 있다. 그러므로 참조형 데이터는 보통 불변하지 않다고 하는 것이다.
<BR>
<BR>
<예>

```
01 obj1 = {
02    a : 1,
03    b: 'bbb'
04 };
05 obj1.a = 2;
```

이 때 프로퍼티 a에 숫자 2를 할당하려고 하면, 데이터 영역에서 2를 찾아 그 주소값을 객체의 변수 영역에 저장한다. 4번째 줄의 명령 전과 후에 변수 obj1이 바라보고 있는 주소는 5001로 변하지 않았다. 문자열이나 숫자를 재할당 했을 때 바라보고 있던 주소가 바뀐 것과 달리 별개의 데이터가 만들어진 것이 아니라 객체 내부의 값만 바뀐 것이다.
